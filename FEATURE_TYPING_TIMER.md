# 功能增强：生成中计时器

## 新增功能

在"生成中"的三个跳动点旁边添加了一个实时计时器，显示等待时间。

## 效果展示

### 视觉效果

```
助手：
┌─────────────────────────┐
│ ⚫ ⚫ ⚫  3s             │  ← 三个点 + 计时器
└─────────────────────────┘
```

### 时间线

```
0s  → 发送消息，显示 ⚫⚫⚫ 0s
1s  → 更新为 ⚫⚫⚫ 1s
2s  → 更新为 ⚫⚫⚫ 2s
3s  → 更新为 ⚫⚫⚫ 3s
...
收到内容 → 三个点和计时器一起消失，显示实际文字
```

## 实现细节

### 1. 计时器启动

在 `sendMessage()` 函数中，发送消息后立即：
- 创建助手消息气泡
- 显示三个跳动的点
- 启动计时器（每秒更新一次）
- 将计时器 ID 存储在元素的 `dataset.timerId` 中

```javascript
let seconds = 0;
const timerInterval = setInterval(() => {
  seconds++;
  timerEl.textContent = `${seconds}s`;
}, 1000);

typingEl.dataset.timerId = timerInterval;
```

### 2. 计时器清除

在以下情况会清除计时器：

**情况 1：收到内容**
- 在 `updateStreamMessage()` 函数中
- 当开始显示实际文字时
- 清除计时器并移除三个点

**情况 2：发送失败**
- 在 `catch` 块中
- 清除计时器并移除整个消息气泡

**情况 3：连接断开**
- 在 `reconnect` 事件中
- 清除计时器（如果存在）

## 优势

1. **透明度** - 用户知道等待了多久
2. **心理预期** - 看到计时器在动，知道系统在工作
3. **性能感知** - 可以直观感受不同任务的响应时间
4. **调试帮助** - 开发者可以看到实际响应时间

## 技术细节

### 计时器精度
- 使用 `setInterval(fn, 1000)` 每秒更新
- 精度为秒级（不显示毫秒）
- 足够满足用户感知需求

### 内存管理
- 每次创建计时器都会存储 ID
- 确保在所有情况下都会清除
- 避免内存泄漏

### 性能影响
- 每秒只更新一次 DOM（一个文本节点）
- 性能影响可忽略不计
- 不会影响消息流式输出

## 测试场景

### 场景 1：正常响应
```
发送消息 → 0s
等待 1 秒 → 1s
等待 2 秒 → 2s
收到内容 → 计时器消失，显示文字
```

### 场景 2：快速响应
```
发送消息 → 0s
立即收到内容 → 计时器消失（可能只显示了 0s）
```

### 场景 3：慢速响应
```
发送消息 → 0s
等待 10 秒 → 10s
等待 20 秒 → 20s
收到内容 → 计时器消失
```

### 场景 4：发送失败
```
发送消息 → 0s
等待 2 秒 → 2s
发送失败 → 计时器清除，消息气泡移除
```

### 计时器样式

- 大小：12px
- 对齐：右对齐
- 最小宽度：30px（确保 "10s" 不会跳动）

### 布局
- 使用 `flexbox` 横向排列
- 三个点和计时器之间间隔 12px
- 垂直居中对齐
