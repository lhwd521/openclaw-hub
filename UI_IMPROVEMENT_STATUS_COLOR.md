# UI 改进：用颜色表示忙碌状态

## 改进说明

将连接的忙碌状态从文字提示改为状态点颜色变化，更加简洁直观。

## 改动对比

### 之前

```
侧边栏：
┌─────────────────────┐
│ ● 数据分析 VPS (忙碌) │  ← 绿点 + 文字
└─────────────────────┘
```

### 现在

```
侧边栏：
┌─────────────────────┐
│ ● 数据分析 VPS       │  ← 黄点（忙碌时）
└─────────────────────┘

┌─────────────────────┐
│ ● 数据分析 VPS       │  ← 绿点（空闲时）
└─────────────────────┘
```

## 状态点颜色说明

| 状态 | 颜色 | 说明 |
|------|------|------|
| 已连接 + 空闲 | 🟢 绿色 | 连接正常，可以发送消息 |
| 已连接 + 忙碌 | 🟡 黄色 | 连接正常，正在处理任务 |
| 连接中 | 🟡 黄色 | 正在建立连接 |
| 重连中 | 🟡 黄色（闪烁） | 正在重新连接 |
| 未连接 | ⚪ 灰色 | 未连接或已断开 |
| 错误 | 🔴 红色 | 连接出错 |

## 实现细节

### 1. sidebar.js 修改

**逻辑变化：**
```javascript
// 之前：直接使用连接状态
<span class="status-dot ${status}"></span>

// 现在：如果已连接且忙碌，显示为 busy
const dotClass = status === "connected" && busy ? "busy" : status;
<span class="status-dot ${dotClass}"></span>
```

**移除文字：**
```javascript
// 之前
<span class="connection-name">${name}${busy ? " (忙碌)" : ""}</span>

// 现在
<span class="connection-name">${name}</span>
```

### 2. style.css 修改

新增 `.status-dot.busy` 样式：

```css
.status-dot.busy {
  background: var(--warning);
}
```

使用 `var(--warning)` 变量（橙黄色），与 `connecting` 状态相同。

## 优势

### 1. 更简洁
- 移除了"(忙碌)"文字
- 连接名称更清爽
- 视觉噪音更少

### 2. 更直观
- 颜色变化一目了然
- 不需要阅读文字
- 符合通用的状态指示习惯

### 3. 国际化友好
- 不依赖文字翻译
- 颜色是通用语言
- 减少翻译工作量

### 4. 空间利用
- 节省横向空间
- 连接名称可以更长
- 布局更紧凑

## 用户体验

### 场景 1：发送消息

```
发送前：
● 总经理 VPS  ← 绿色

发送后：
● 总经理 VPS  ← 立即变黄色

收到回复：
● 总经理 VPS  ← 变回绿色
```

### 场景 2：多个连接

```
● 数据分析 VPS    ← 绿色（空闲）
● 财务顾问 VPS    ← 黄色（忙碌）
● 编剧助手 VPS    ← 绿色（空闲）
⚪ 项目经理 VPS   ← 灰色（未连接）
```

一眼就能看出哪些连接在工作。

## 技术实现

### 状态判断逻辑

```javascript
const status = state.connectionStatuses[conn.id];  // "connected" | "disconnected" | ...
const busy = state.isBusy[conn.id];                // true | false

// 优先级：
// 1. 如果已连接且忙碌 → "busy" (黄色)
// 2. 否则使用原始状态 → "connected" (绿色) | "disconnected" (灰色) | ...
const dotClass = status === "connected" && busy ? "busy" : status;
```

### CSS 变量

使用主题变量确保一致性：

```css
--success: #4caf50;   /* 绿色 - 成功/空闲 */
--warning: #ff9800;   /* 橙黄色 - 警告/忙碌 */
--error: #f44336;     /* 红色 - 错误 */
--text-muted: #5c6070; /* 灰色 - 未连接 */
```

## 可访问性

虽然使用颜色表示状态，但仍然保持了良好的可访问性：

1. **颜色对比度**
   - 绿色、黄色、红色都有足够的对比度
   - 在白色/深色背景上都清晰可见

2. **状态点大小**
   - 8x8 像素，足够大
   - 容易注意到

3. **位置一致**
   - 状态点始终在连接名称左侧
   - 位置固定，易于查找

4. **辅助信息**
   - 鼠标悬停可以看到完整信息（未来可添加 tooltip）
   - 在线用户数量提供额外上下文
