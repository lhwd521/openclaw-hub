# 可视化组织架构图 - 使用指南

## 功能概述

这是一个**拖拽式的组织架构编辑器**，让你直观地设置 OpenClaw 之间的上下级关系。就像公司的组织架构图一样，你可以设置谁管理谁。

## 操作步骤

### 第一步：创建根节点（最高层级）

1. 从左侧"可用连接"区域，找到你想作为最高层级的连接
2. **按住鼠标左键拖拽**到右侧空白画布
3. 松开鼠标，节点创建成功

**示例：**
```
拖拽 "总经理 VPS" → 右侧画布
结果：创建了一个根节点
```

### 第二步：添加下级

1. 从左侧拖拽另一个连接
2. **拖到已有节点上**（节点会高亮显示）
3. 松开鼠标，建立上下级关系

**示例：**
```
拖拽 "数据分析 VPS" → "总经理 VPS" 节点上
结果："数据分析 VPS" 成为 "总经理 VPS" 的下级
```

### 第三步：设置职责描述

1. 点击节点上的 **✏️ 编辑按钮**
2. 在弹出的对话框中输入职责描述
3. 点击确定

**示例：**
```
总经理 VPS → 职责：统一管理和协调所有团队成员
数据分析 VPS → 职责：数据分析、网络爬虫、数据可视化
财务顾问 VPS → 职责：个人财务分析、投资建议、预算规划
```

### 第四步：保存架构

点击右上角的 **[保存架构]** 按钮

## 实际使用场景

### 场景 1：简单的两层架构

```
        总经理 VPS
        (统一管理)
            |
    +-------+-------+-------+
    |       |       |       |
  数据    财务    编剧    开发
  分析    顾问    助手    工程师
```

**操作步骤：**
1. 拖拽"总经理 VPS"到画布 → 创建根节点
2. 拖拽"数据分析 VPS"到"总经理 VPS"上 → 成为下级
3. 拖拽"财务顾问 VPS"到"总经理 VPS"上 → 成为下级
4. 拖拽"编剧助手 VPS"到"总经理 VPS"上 → 成为下级
5. 拖拽"开发工程师 VPS"到"总经理 VPS"上 → 成为下级
6. 为每个节点设置职责描述
7. 保存架构

### 场景 2：三层架构（CEO → 经理 → 员工）

```
            CEO VPS
         (最高决策)
              |
        +-----+-----+
        |           |
    技术经理      业务经理
        |           |
    +---+---+   +---+---+
    |       |   |       |
  前端    后端  销售    客服
  开发    开发
```

**操作步骤：**
1. 拖拽"CEO VPS"到画布 → 根节点
2. 拖拽"技术经理 VPS"到"CEO VPS"上 → CEO 的下级
3. 拖拽"业务经理 VPS"到"CEO VPS"上 → CEO 的下级
4. 拖拽"前端开发 VPS"到"技术经理 VPS"上 → 技术经理的下级
5. 拖拽"后端开发 VPS"到"技术经理 VPS"上 → 技术经理的下级
6. 拖拽"销售 VPS"到"业务经理 VPS"上 → 业务经理的下级
7. 拖拽"客服 VPS"到"业务经理 VPS"上 → 业务经理的下级
8. 设置每个节点的职责
9. 保存架构

### 场景 3：多个独立的团队

```
    团队A领导          团队B领导
        |                  |
    +---+---+          +---+---+
    |       |          |       |
  成员1   成员2      成员3   成员4
```

**操作步骤：**
1. 拖拽"团队A领导"到画布 → 第一个根节点
2. 拖拽"团队B领导"到画布（不同位置）→ 第二个根节点
3. 分别为两个团队添加成员
4. 保存架构

## 工作原理

### 当你向上级发送消息时

假设你的架构是：

```
总经理 VPS
    |
    +-- 数据分析 VPS (职责：数据分析、网络爬虫)
    +-- 财务顾问 VPS (职责：财务分析、投资建议)
```

**你向"总经理 VPS"发送：**
```
帮我分析特斯拉股票
```

**系统自动构建的完整消息：**
```
[系统信息]
你是总经理，负责：统一管理和协调所有团队成员

你的直接下属：
1. 数据分析 VPS (在线)
   职责：数据分析、网络爬虫

2. 财务顾问 VPS (在线)
   职责：财务分析、投资建议

---

用户任务：帮我分析特斯拉股票

如果需要下属协助，请返回 JSON 格式：
{
  "delegate": true,
  "tasks": [
    {"subordinate": "数据分析 VPS", "task": "具体任务"},
    {"subordinate": "财务顾问 VPS", "task": "具体任务"}
  ]
}

如果你自己能完成，直接回答即可。
```

### 总经理 AI 的决策

总经理 OpenClaw 会分析任务，然后决定：

**选项 1：自己完成**
```
直接回答用户的问题
```

**选项 2：分配给下属**
```json
{
  "delegate": true,
  "tasks": [
    {
      "subordinate": "数据分析 VPS",
      "task": "抓取特斯拉最近3个月的股票数据"
    },
    {
      "subordinate": "财务顾问 VPS",
      "task": "基于数据给出投资建议"
    }
  ]
}
```

### 系统执行分配

前端解析 JSON 后：

1. 连接"数据分析 VPS"，发送任务
2. 连接"财务顾问 VPS"，发送任务
3. 等待结果
4. 将结果汇总发送给"总经理 VPS"
5. 总经理给出最终答案

## 高级功能

### 多层级管理

支持无限层级：

```
CEO
 └─ 副总裁
     └─ 总监
         └─ 经理
             └─ 主管
                 └─ 员工
```

每一层都可以向下分配任务。

### 移除节点

1. 鼠标悬停在节点上
2. 点击右上角的 **×** 按钮
3. 确认删除

**注意：** 删除节点会同时删除它的所有下级节点！

### 重置架构

点击右上角的 **[重置]** 按钮，清空整个组织架构，所有连接回到"可用连接"池。

### 编辑已有节点

- 点击 **✏️** 按钮修改职责描述
- 点击 **➕** 按钮快速添加下级（会提示从可用连接中选择）

## 视觉提示

### 节点状态

- **绿点 ●** - 连接在线
- **灰点 ○** - 连接离线
- **蓝色左边框** - 有下级的节点
- **高亮边框** - 拖拽时的放置目标

### 连接线

- **竖线** - 表示上下级关系
- **横线** - 连接到具体节点

### 拖拽反馈

- **拖拽中** - 卡片半透明
- **可放置** - 目标节点高亮绿色
- **放置成功** - 自动刷新显示新架构

## 常见问题

### Q: 可以有多个根节点吗？

A: 可以！你可以创建多个独立的团队，每个团队有自己的领导。

### Q: 可以跨层级分配任务吗？

A: 目前只能分配给直接下属。如果需要跨层级，可以让中间层级转发任务。

### Q: 如果下属离线了怎么办？

A: 系统只会将任务分配给在线的下属。离线的下属会被自动跳过。

### Q: 可以一个节点有多个上级吗？

A: 目前不支持。每个节点只能有一个直接上级（符合传统组织架构）。

### Q: 职责描述有什么用？

A: 职责描述会发送给上级 AI，帮助它决定应该把任务分配给谁。描述越详细，分配越准确。

### Q: 可以导出组织架构图吗？

A: 目前架构保存在浏览器本地存储中。未来可以添加导出为图片或 JSON 的功能。

## 最佳实践

### 1. 清晰的职责描述

**好的描述：**
```
数据分析 VPS
职责：
- 网络爬虫和数据抓取
- 数据清洗和预处理
- 数据可视化和报表生成
- 支持的数据源：股票、新闻、社交媒体
```

**不好的描述：**
```
数据分析 VPS
职责：做数据相关的事情
```

### 2. 合理的层级设计

**推荐：** 2-3 层
```
总经理 → 部门经理 → 专员
```

**不推荐：** 过深的层级
```
CEO → VP → 总监 → 经理 → 主管 → 组长 → 员工
```
（层级太深会导致任务传递效率低）

### 3. 平衡的团队规模

**推荐：** 每个管理者管理 3-7 个下属

**不推荐：** 一个管理者管理 20 个下属（太难协调）

### 4. 专业化分工

让每个 OpenClaw 专注于特定领域，避免职责重叠。

## 技术实现

### 数据结构

```javascript
{
  nodes: {
    "conn-1": {
      id: "conn-1",
      role: "统一管理和协调",
      subordinates: ["conn-2", "conn-3"],
      manager: null  // 根节点
    },
    "conn-2": {
      id: "conn-2",
      role: "数据分析",
      subordinates: [],
      manager: "conn-1"
    }
  },
  roots: ["conn-1"]  // 根节点列表
}
```

### 保存位置

- **浏览器本地存储：** `localStorage["openclaw-hub.org-structure"]`
- **应用状态：** `store.state.orgStructure`

### 拖拽实现

使用原生 HTML5 Drag and Drop API：
- `draggable="true"` - 使元素可拖拽
- `dragstart` - 开始拖拽
- `dragover` - 拖拽经过
- `drop` - 放置
