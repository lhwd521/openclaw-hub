# OpenClaw 智能角色管理和任务路由系统

## 功能概述

这是一个让多个 OpenClaw 实例协同工作的系统。你可以为每个 OpenClaw 分配专业角色，然后让"项目经理"角色智能地将任务分配给合适的专家。

## 核心概念

### 1. 角色系统

每个 OpenClaw 连接可以被赋予一个专业角色：

- **📊 数据分析师** - 数据分析、网络爬虫、数据可视化
- **💰 财务顾问** - 个人财务分析、投资建议、预算规划
- **🎬 编剧助手** - 剧本创作、故事构思、角色设计
- **👔 项目经理** - 任务协调、团队管理、进度跟踪
- **💻 开发工程师** - 代码编写、调试、技术方案设计
- **🎨 设计师** - UI/UX 设计、视觉创意、原型制作
- **🔬 研究员** - 学术研究、文献综述、实验设计
- **🌐 翻译** - 多语言翻译、本地化、文化适配
- **⚙️ 自定义角色** - 你可以定义任何专业领域

### 2. 智能任务路由

当你向"项目经理"角色发送任务时，它会：

1. **分析任务** - 理解任务需要哪些专业能力
2. **制定方案** - 决定分配给哪些专家，是并行还是顺序执行
3. **分发任务** - 将子任务发送给对应的专家
4. **汇总结果** - 整合所有专家的工作成果，给出最终答案

## 使用步骤

### 第一步：添加 OpenClaw 连接

1. 进入"连接管理"页面
2. 添加你的多个 OpenClaw 实例
3. 确保它们都能成功连接

示例配置：
```
连接 1: "数据分析 VPS"  - https://data-analyst.trycloudflare.com
连接 2: "财务顾问 VPS"  - https://finance-advisor.trycloudflare.com
连接 3: "编剧助手 VPS"  - https://scriptwriter.trycloudflare.com
连接 4: "项目经理 VPS"  - https://project-manager.trycloudflare.com
```

### 第二步：分配角色

1. 进入"角色管理"页面（新增的导航菜单）
2. 你会看到所有连接的卡片
3. 从下方的"角色模板"区域，**拖拽**角色图标到对应的连接卡片上
4. 点击"保存配置"

**拖拽操作示例：**
```
1. 鼠标按住 "📊 数据分析师" 图标
2. 拖动到 "数据分析 VPS" 的卡片上
3. 松开鼠标，角色就分配成功了
```

**重要提示：**
- 至少要有一个"项目经理"角色
- 其他角色根据你的需求配置
- 可以有多个相同角色的连接（负载均衡）

### 第三步：使用智能路由

1. 进入"聊天"页面
2. 选择你分配了"项目经理"角色的连接
3. 在聊天界面右上角，你会看到"🤖 智能路由"开关
4. 打开这个开关
5. 现在发送任务，项目经理会自动分配给团队成员

## 实际使用场景

### 场景 1：股票投资分析

**你的团队配置：**
- 连接 1: 数据分析师
- 连接 2: 财务顾问
- 连接 3: 项目经理

**你向项目经理发送：**
```
帮我分析一下特斯拉（TSLA）最近的股票走势，
并给出投资建议。
```

**项目经理的工作流程：**

1. **分析任务** → 识别需要数据分析和财务建议
2. **制定方案** → 并行执行
   - 数据分析师：抓取特斯拉股票数据，分析走势
   - 财务顾问：基于数据给出投资建议
3. **分发任务** → 同时向两个专家发送子任务
4. **汇总结果** → 整合数据和建议，生成完整报告

**你会看到：**
```
📋 任务分配方案
执行策略：🔀 并行执行
分配理由：需要数据分析和财务建议两个专业领域

1. 数据分析师
   任务：抓取特斯拉股票数据并分析走势
   预计：3-5分钟

2. 财务顾问
   任务：基于数据给出投资建议
   预计：2-3分钟

---

✅ 数据分析师 (完成 - 4.2s)
任务：抓取特斯拉股票数据并分析走势
结果：[数据分析师的详细分析...]

✅ 财务顾问 (完成 - 3.8s)
任务：基于数据给出投资建议
结果：[财务顾问的投资建议...]

---

📊 项目经理总结
[整合后的完整投资报告...]
```

   - 编剧助手：基于研究结果创作剧本

## 高级功能

### 自定义角色

如果预设角色不满足需求，可以创建自定义角色：

1. 拖拽"⚙️ 自定义角色"到连接卡片
2. 输入角色名称，例如："法律顾问"
3. 输入专业领域描述：
   ```
   专业领域：合同审查、法律咨询、风险评估
   能力：熟悉商业法、知识产权法、劳动法
   ```
4. 保存配置

### 多个相同角色

你可以配置多个相同角色的连接，实现负载均衡：

```
连接 1: 数据分析师 A
连接 2: 数据分析师 B
连接 3: 数据分析师 C
连接 4: 项目经理
```

当项目经理需要数据分析时，会自动选择一个可用的数据分析师。

### 查看角色信息

在聊天界面，连接名称旁边会显示角色徽章：

```
[📊 数据分析师] 数据分析 VPS
[👔 项目经理] 项目经理 VPS
```

## 技术原理

### 项目经理如何知道有哪些"小弟"？

1. 系统会读取所有已连接的 OpenClaw
2. 过滤出有角色分配且在线的连接
3. 将角色列表发送给项目经理
4. 项目经理使用 AI 分析任务并选择合适的角色

### 任务分配的 AI 提示词

项目经理收到的提示词示例：

```
你是一个项目经理，负责将用户任务分配给合适的专家团队成员。

可用的专家团队：
- 数据分析 VPS (数据分析师): 数据分析、网络爬虫、数据可视化
- 财务顾问 VPS (财务顾问): 个人财务分析、投资建议、预算规划
- 编剧助手 VPS (编剧助手): 剧本创作、故事构思、角色设计

用户任务：
帮我分析一下特斯拉最近的股票走势，并给出投资建议。

请分析这个任务，并返回 JSON 格式的任务分配方案...
```

### 执行策略

**并行执行（parallel）：**
- 多个子任务可以同时进行
- 适合独立的任务
- 速度最快

**顺序执行（sequential）：**
- 后一个任务依赖前一个的结果
- 适合有依赖关系的任务
- 保证逻辑正确

**单人执行（single）：**
- 只需要一个专家
- 最简单的情况

## 常见问题

### Q: 必须要有项目经理吗？

A: 不是必须的。如果不使用智能路由功能，可以直接和任何角色的 OpenClaw 聊天。项目经理只在需要多角色协作时才必要。

### Q: 可以手动选择专家吗？

A: 目前是由项目经理 AI 自动分配。未来可以添加手动模式。

### Q: 如果某个专家失败了怎么办？

A: 项目经理会在最终报告中说明哪个任务失败了，并给出建议。

### Q: 成本会增加吗？

A: 是的。多个 OpenClaw 同时工作会消耗更多 API 调用。但并行执行可以节省时间。

### Q: 可以中途取消吗？

A: 可以点击"中断"按钮，但已经发送给专家的任务可能无法立即停止。
