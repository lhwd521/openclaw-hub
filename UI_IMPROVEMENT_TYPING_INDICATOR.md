# UI 优化：生成中提示位置调整

## 问题

之前"生成中..."的提示显示在输入框下面，位置不太合理，用户体验不好。

## 解决方案

将"生成中..."提示移到助手消息气泡内部，更加直观和自然。

## 改动说明

### 1. 移除输入框下方的状态栏

**之前：**
```
┌─────────────────────────┐
│ [聊天消息区域]          │
└─────────────────────────┘
┌─────────────────────────┐
│ [输入框]                │
│ ⏳ 生成中...  ← 这里    │
└─────────────────────────┘
```

**现在：**
```
┌─────────────────────────┐
│ [聊天消息区域]          │
│                         │
│ 助手：                  │
│ ┌─────────────────────┐ │
│ │ ⚫⚫⚫ ← 这里        │ │
│ └─────────────────────┘ │
└─────────────────────────┘
┌─────────────────────────┐
│ [输入框]                │
└─────────────────────────┘
```

### 2. 修改的文件

#### chat.js
- 移除 `statusEl` 的引用
- 移除 HTML 中的 `<div class="chat-status">`
- 在消息气泡内显示 typing indicator
- 清理不再需要的 `statusEl.innerHTML = ""` 代码

#### style.css
- 移除 `.chat-status` 样式
- 保留原有的 `.typing-indicator` 样式（用于其他地方）
- 新增 `.typing-indicator-inline` 样式（用于消息气泡内）

### 3. 新的 Typing Indicator 样式

```css
.typing-indicator-inline {
  display: inline-flex;
  gap: 4px;
  padding: 8px 0;
}

.typing-indicator-inline span {
  width: 8px;
  height: 8px;
  background: var(--text-secondary);
  border-radius: 50%;
  animation: typing 1.4s infinite;
}
```

**效果：** 三个小圆点在消息气泡内跳动

### 4. 工作流程

1. 用户发送消息
2. 系统创建一个空的助手消息气泡
3. 在气泡内显示跳动的三个点 `⚫⚫⚫`
4. AI 开始返回内容时，替换为实际文本
5. 文本逐步更新（流式输出）
6. 完成后，气泡显示完整内容

### 5. 视觉效果

**生成中：**
```
┌─────────────────────────┐
│ 助手                    │
│ ┌─────────────────────┐ │
│ │ ⚫ ⚫ ⚫             │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

**生成中（有内容）：**
```
┌─────────────────────────┐
│ 助手                    │
│ ┌─────────────────────┐ │
│ │ 根据您的要求，我来  │ │
│ │ 分析一下...         │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

**完成：**
```
┌─────────────────────────┐
│ 助手                    │
│ ┌─────────────────────┐ │
│ │ 根据您的要求，我来  │ │
│ │ 分析一下特斯拉的股  │ │
│ │ 票走势...           │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```

## 优势

1. **更直观** - 用户一眼就能看到 AI 正在回复
2. **位置合理** - 提示在消息气泡内，不占用额外空间
3. **视觉连贯** - 从"生成中"到"显示内容"是平滑过渡
4. **符合习惯** - 类似微信、Telegram 等聊天应用的设计

## 测试

启动应用后：
1. 发送一条消息
2. 观察助手消息气泡
3. 应该看到三个跳动的小圆点
4. 内容开始出现时，圆点消失，文字逐步显示

## 兼容性

- 所有现代浏览器支持
- CSS 动画流畅
- 不影响其他功能
